# Brand Page (Web Portfolio) Project Rules & Patterns

## 🛑 Failure Patterns (절대 반복 금지)
1. **노션(Notion) 스타일의 좁은 너비 사용 금지**: 상세 페이지에서 `maxWidth: 700px` 같은 속성으로 중앙 정렬된 좁은 문서를 띄우는 행위는 최악의 실패 패턴임. 항상 뷰포트를 시원하게 쓰는 와이드(Wide) 애플 스타일을 유지할 것.
2. **페이지 이동 시 레이아웃(양식) 파괴 금지**: 메인 페이지에서 상세 페이지로 이동할 때, 2분할(Split) 레이아웃이 깨지고 1단 레이아웃으로 변하면 안 됨. **좌측(프로필/경력)은 고정된 채 우측 패널만 교체되는 구조**를 무조건 유지할 것.
3. **콘텐츠 누락 및 축약 금지 (디자인만 흉내 내기 금지)**: 노션 이력서 원본에 있는 디테일한 텍스트(기획 배경, 핵심 성과 등)를 임의로 축약하지 말 것. 원본 텍스트는 토씨 하나 틀리지 않고 100% 이관하되 디자인 요소(CSS)만 애플 스타일로 입힐 것.
4. **상업적 랜딩페이지 룩 금지**: 전형적인 상업 페이지나 벤토 박스(Bento Box) 스타일 지양. 극대화된 여백(Margin)과 대형 폰트를 사용할 것.
5. **이미지 비율 왜곡 (깨짐) 금지**: `object-fit: cover` 없이 이미지를 끼워 넣어 비율이 찌그러지는 현상 차단.

## 🟢 Success Patterns (항상 유지해야 할 사항)
1. **글로벌 2분할 플로팅(App-Window) 레이아웃**: 좌우 10% 여백을 주어 화면 한가운데에 떠 있는 거대한 앱처럼 보이게 하는 디자인.
2. **Apple-style Micro-interactions**: 스크롤 시 부드럽게 떠오르는 `Fade-Up Reveal` 애니메이션과 썸네일 Hover 스케일.
